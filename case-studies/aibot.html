What I Learnt

Prompt Engineering

Crafting effective system prompts and managing context windows taught me how to guide LLM behavior reliably. Few-shot examples significantly improve output consistency.

Streaming Responses

Implementing token-by-token streaming via WebSockets dramatically improves perceived latency. Users see responses appear in real-time rather than waiting for complete generation.

Function Calling Architecture

Building a modular function calling system allowed the AI to perform real actions like searching, calculating, and fetching data. Schema validation is crucial for reliability.

Token Management & Costs

Learned to carefully manage context length, implement conversation summarization, and cache repeated queries to control API costs at scale.

Conclusion

Building an AI chat assistant opened my eyes to the fascinating world of large language models. The balance between capability and cost, combined with the need for reliable outputs, presents unique engineering challenges.

The streaming implementation was particularly rewarding - seeing responses flow character by character creates a much more engaging user experience than waiting for complete responses.

I've gained deep appreciation for the importance of prompt engineering and the subtle art of guiding AI behavior through careful instruction design.

Thank you for giving your time.
