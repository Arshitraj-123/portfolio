What I Learnt

Server Components in Next.js 13+

Working with the new App Router taught me how to leverage server components for better performance. Data fetching on the server reduces client-side JavaScript and improves initial page load times significantly.

Optimistic UI Updates

For actions like adding items to cart, I implemented optimistic updates that instantly reflect in the UI while the server processes the request in the background. This makes the app feel incredibly responsive.

Stripe Integration Best Practices

Integrating Stripe for payments taught me about webhooks, handling payment intents securely, and the importance of idempotency keys for preventing duplicate charges.

Image Optimization

Using Next.js Image component with proper sizing, lazy loading, and blur placeholders significantly improved the Core Web Vitals scores, especially LCP.

Conclusion

Building ShopStreet was an incredible learning experience. The combination of Next.js, MongoDB, and Firebase proved to be a powerful stack for building scalable e-commerce applications. The App Router's server components allowed for excellent performance while maintaining a great developer experience.

I learned the importance of planning database schemas carefully before implementation. Having clear relationships between Users, Products, Orders, and Stores made the development process much smoother and the codebase more maintainable.

The most challenging part was implementing the real-time inventory system to prevent overselling. Using MongoDB transactions and optimistic locking helped solve this issue elegantly.

Thank you for taking the time to read about this project.
